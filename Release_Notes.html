Purpose

The X-CUBE-CELLULAR (STM32 Cellular software expansion for STM32Cube)

  • Enables connectivity over Cellular Network. The network access technology depends on the Cellular Modem used. It can be LTE CatM, NBIoT or fallback in 2G.
  • Provides for STM32CubeMX, application example that connect to internet.
  • This example, same for every supported hardware setups, demonstrates the cellular middleware capabilities.
  • It contains 1 or 2 instances of Echo, Ping usage and throughput display.
  • For B-L462E-CELL1 it also provides an application that read sensors and display information (sensors/network) on the LCD display.
  • This example provides a vertical application for this all in one IoT board.

New versions with more advanced features and optimization will be released regularly.

Please take time to read at least the Release Notes and the Cellular MCU Wiki.

The Cellular middleware is compliant with MISRA-C®:2012 with some minor limitations and checked against a static analysis tool to eliminate run-time errors. Reports are available on demand under NDA.

Backward compatibilities

X-CUBE-CELLULAR APIs are no backward compatible. Indeed APIs over Data Cache has been developped and merged to Cellular management ones. COM APIs are still backward compatible. So with V6.0.0 onwards there are 2 APIs for applications, COM (Data plane) and Ctrl (Control plane)
